Log poisoning

SSH

Conf is located in:

  • /etc/ssh/sshd_config

Logs can be located in:

Path
/var/log/auth.log
/var/log/secure


ProFTPd

Conf is located in:

  • /etc/proftpd/proftpd.conf

Logs can be located in:

Path
/var/log/proftpd/access.log
/var/log/proftpd/ftp.log


vsFTPd

Conf is located in:

  • /etc/vsftpd.conf

Logs can be located in:

Path
/var/log/vsftpd.log
/var/log/vsftpd/vsftpd_access.log
/var/log/vsftpd/xferlog


Apache

Conf can be located in:

  • /etc/apache2/apache.conf
  • /etc/apache2/apache2.conf
  • /etc/httpd/conf/httpd.conf
  • /etc/httpd/conf.d/apache.conf
  • /etc/httpd/conf/extra/apache.conf

Access logs can be located in:

Path
/var/log/apache2/access.log
/var/log/apache2/access_log
/var/log/httpd/access.log
/var/log/httpd/access_log

Error logs can be located in:

Path
/var/log/apache2/error.log
/var/log/httpd/error_log
/var/log/httpd/error_log
/var/log/apache2/error_log

Proc files

/proc/self/cwd/

Symbolic link that represents the current working directory of the process accessing it. Useful in some CTF where you can try to include /proc/self/cwd/flag.txt.

/proc/self/cmdline

Contains the command line arguments that were used to start the process accessing it.

/proc/self/comm

Contains the name of the current executable (the command that started the process)

/proc/self/environ

Contains the environment variables of the process accessing it.

/proc/self/status

Contains various details about the process, including the UID/GID of the user running the process

Interesting files

/etc/passwd

Contains user account information: username, UID, GID, home directory, and the shell used by each user.

Shell history

1

/home/${USER}/.${CONNECTION_SHELL}_history

You can find ${USER} and ${CONNECTION_SHELL} in /etc/passwd

Private SSH keys

1

/home/${USER}/.ssh/id_rsa

You can find ${USER} in /etc/passwd

/var/log/syslog

Contains general system log messages, including information about system events, services, and applications

/etc/crontab

Contains system-wide cron job definitions along with the user under which they should execute.

Misc

Filter base64

1

php://filter/convert.base64-encode/resource=FILENAME

# to bypass restrictions

Sometimes you can use the # symbol to bypass verification:

1
2

?filename=file:///etc/passwd#https
?filename=http://mywebsite.com/#https
